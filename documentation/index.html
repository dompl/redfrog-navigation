1. Introduction

Using this documentation guide you can build your navigation "step by step". Please read this carefully to avoid misleads and errors and feel free to contact the author for bugs or problems with the installation.

2. Installation

To start building your navigation element you need to include the CSS file to your <head> section. 

<link href="css/navigation.css" rel="stylesheet">

The JS files can be placed inside the <head> or before the tag </body>. 

<script type="text/javascript" src="js/jquery.js"></script>
<script type="text/javascript" src="js/navigation.js"></script>
Be careful with your folders. Here we are using a folder named css for the stylesheets and a folder named js for the javascript files.
The navigation works with jQuery 1.9 up to jQuery 3.

Let's start with a very basic markup: 

<nav id="navigation" class="navigation">
    
    <div class="nav-header">
        <div class="nav-toggle"></div>
    </div>
	
    <div class="nav-menus-wrapper">
        <ul class="nav-menu">
            <li class="active"><a href="#">Home</a></li>
            <li><a href="#">Services</a></li>
            <li><a href="#">Blog</a></li>
            <li><a href="#">Contact</a></li>
        </ul>
    </div>
	
</nav>

We have three main elements:

  • .navigation: it's the main wrapper of the navigation. It must has an ID to identify the navigation in the DOM.
  • .nav-header: It's the header of the navigation. It holds the .nav-brand or the .nav-logo and .nav-toggle.
  • .nav-menus-wrapper: It's the wrapper of the menu items and the "offcanvas" panel on mobile devices.

Now you can initialize the navigation: 

$(document).ready(function(){
    $("#navigation").navigation();
});

The class active on the menu item indicates the active page.

3. Menus

The first level menus can be aligned to left, right or placed in the center. The default alignment is to left. To align the menu to right or center just use the respectively classes align-to-right and nav-menu-centered: 

<!-- Aligned to left (default) -->
<ul class="nav-menu">
    <li class="active"><a href="#">Home</a></li>
    <li><a href="#">Services</a></li>
    <li><a href="#">Blog</a></li>
    <li><a href="#">Contact</a></li>
</ul>

<!-- Aligned to right -->
<ul class="nav-menu align-to-right">
    <li class="active"><a href="#">Home</a></li>
    <li><a href="#">Services</a></li>
    <li><a href="#">Blog</a></li>
    <li><a href="#">Contact</a></li>
</ul>

<!-- Placed in the center -->
<ul class="nav-menu nav-menu-centered">
    <li class="active"><a href="#">Home</a></li>
    <li><a href="#">Services</a></li>
    <li><a href="#">Blog</a></li>
    <li><a href="#">Contact</a></li>
</ul>

Centered menus can't be conjugated with other menus. You can use only one centered menu per navigation.
The centered menu considers the width of the other elements inside the navigation bar (buttons, text, brand text, logo) to calculate the distance of the left and right borders.
Menus with left or right alignment can be placed in the same navigation bar. You can use as many as you want.

Using menus aligned to left and right at same time: 

<nav id="navigation" class="navigation">

    <div class="nav-header">
        <div class="nav-toggle"></div>
    </div>

    <div class="nav-menus-wrapper">
        <ul class="nav-menu">

            <li><a href="#">Services</a></li>
            <li><a href="#">Blog</a></li>
        </ul>
        <ul class="nav-menu align-to-right">
            <li><a href="#">Team</a></li>
            <li><a href="#">Portfolio</a></li>
            <li><a href="#">Contact</a></li>
        </ul>
    </div>

</nav>

13. Callbacks

See how to use the 3 callback functions available:

onInit
Executes a code when the navigation is initialized. 

$("#navigation").navigation({
    onInit: function() {
        // your code
    }
});

onShowOffCanvas
Executes a code when the offcanvas panel is shown. 

$("#navigation").navigation({
    onShowOffCanvas: function() {
        // your code
    }
});

onHideOffCanvas
Executes a code after the offcanvas panel is hidden. 

$("#navigation").navigation({
    onHideOffCanvas: function() {
        // your code
    }
});

14. Methods

See the available methods to interact with the navigation:

toggleOffcanvas()
Show or hide the offcanvas panel. 

$("#navigation").data("navigation").toggleOffcanvas();

toggleSearch()
Show or hide the search panel. 

$("#navigation").data("navigation").toggleSearch();

15. Fixed navigation

For more compatibility we provide an additional script to make the navigation fixed. Include the navigation.fixed.js in your project after navigation.js. Using this additional script you can make the navigation fixed on a desired position at the top of your page. 

<script type="text/javascript" src="js/navigation.fixed.js"></script>

// navigation fixed at the 10px from top
$("#navigation").fixed({
    topSpace: 10
});

Options:

Option Default value Type Description
topSpace 0 Integer Pixels between the page's top and the navigation's top
placeholder true Boolean Inserts a fake element with the same navigation height to avoid the content jumps up when the navigation goes to a fixed position
onFixed -- Custom code (callback) Executes your custom code when the navigation gets fixed position
onStatic -- Custom code (callback) Executes your custom code when the navigation gest static position

The plugin adds a wrapper with the class navigation-fixed-wrapper. You can apply your styles to it, for example, border and background color.
The fixed navigation gets the width of his parent container. So, all paddings and borders from parent container will be considered.

16. Scrollspy behavior

It is possible to automatically change the active item based on the scroll position. Add the file navigation.scrollspy.js to your project after navigation.js. 

<script type="text/javascript" src="js/navigation.scrollspy.js"></script>

// changes the active link 20px before enter the next area
$("#navigation").scrollspy({ 
    offset: 20 
});

Options:

Option Default value Type Description
activeClass "active" String The class that gets added to the li element of the menu
scrollSpeed 500 (ms) Integer The scrolling speed
offset 0 Integer The offset distance
onChange -- Custom code (callback) Executes your custom code when the navigation goes to the next item
onExit -- Custom code (callback) Executes your custom code when the navigation goes outside of the spied area

We do not recommend the use of scrollspy with two menus in the same navigation bar. This may cause errors because the plugin searches for a main list of menu items.

17. Main CSS classes

The navigation works with two main CSS classes according to the variable "mobileBreakpoint". These classes indicate the navigation mode. The CSS classes are:

  • navigation-portrait: this class will be applied to indicate that navigation is in the portrait mode. It happens when the device screen width has a size minor/equal than value of "mobileBreakpoint".

  • navigation-landscape: this class will be applied to indicate that navigation is in the landscape mode. It happens when the device screen width has a size greater than value of "mobileBreakpoint".

The navigation uses these classes to apply specifics styles and behaviors depending of your device screen size. You can use these CSS classes to apply different styles to different devices. Take a look on the examples:

Changing the background color of the navigation on large devices: 

.navigation-landscape .navigation{
    background-color: black;
}

Changing the menu item font on small devices: 

.navigation-portrait .nav-menu > li > a{
    font-weight: bold;
}
